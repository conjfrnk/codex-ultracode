# Companion Runtime

Use the companion runtime only when execution, resumability, scheduling, evidence capture, or cloud task control materially helps the user's request. Direct answers and focused Mode 2 edits do not need it.

The release bundle includes `conductor-runtime.pyz`, a dependency-free Python zip application. A local installation normally places it at `${CODEX_CONDUCTOR_HOME:-$HOME/.codex/conductor}/bin/conductor-runtime.pyz`. Prefer a `conductor-runtime` command on `PATH`; otherwise invoke:

```text
python3 "${CODEX_CONDUCTOR_HOME:-$HOME/.codex/conductor}/bin/conductor-runtime.pyz" <command> ...
```

An extracted release bundle also includes `install.py` and `release-manifest.json`. Run `python3 install.py --dry-run` first, then `python3 install.py --allow-writes --approve conductor-install` to install hash-verified versioned and active copies outside repositories. A differing existing installation additionally requires `--replace --approve conductor-update`; repeated identical installation is a no-write success. Installation never edits PATH, shell startup, services, or schedulers.

When this Skill is loaded from the `codex-conductor` plugin and no active external runtime exists, the plugin embeds the same packaged runtime at `../../scripts/conductor-runtime.pyz` relative to the Skill root. Resolve that path from the Skill locator, require it to be a regular non-symlink file, and invoke it with `python3`; never copy it into a repository. Plugin and standalone runtime state use the same external `${CODEX_CONDUCTOR_HOME:-~/.codex/conductor}` root.

When developing from the source repository, the equivalent command is:

```text
python3 -m conductor_runtime <command> ...
```

`doctor` checks both the `codex` launcher and the sibling `codex-code-mode-host` required by code-mode-only models. A launcher symlink without its companion is unhealthy. With explicit `--repair-codex-host --approve codex-host-repair`, it may create only the missing sibling symlink inside the current user's home; it never overwrites a broken, stale, mismatched, or non-executable entry. After an explicit missing-host error, do not retry that transport in the same session.

Runtime state belongs outside repositories. The default is:

```text
${CODEX_CONDUCTOR_HOME:-~/.codex/conductor}/workspaces/<workspace-name>-<hash>/
```

Useful flows:

- `auto` is the preferred high-level entry point for substantial Mode 3–5 work when runtime model calls are proportionate and disclosed. Its serial default compiles one direct bounded Codex step without a planning call. Omitted direct ceilings scale by `low|medium|high|xhigh|ultra`: 4k/8k/12k/16k/20k read-only and 8k/16k/24k/32k/40k write; an explicit cap remains authoritative. With an explicit verifier and multi-worker envelope, it tries that direct worker once and opens a same-session model planner only after failed verification; later attempts may use the approved envelope. Omitted `--planner-effort` resolves to `medium` only for bounded verified work without broad or sensitive complexity markers and otherwise resolves to `high`; current v5 permits short refactors, migrations, bounded test additions, and bounded idempotency repair when no independent hard marker remains, while a framework name, exact production-code non-modification guard, refactoring alone, or idempotency alone is not a hard signal. Explicit multi-file/multi-module scope and all existing broad or sensitive markers remain high. Explicit effort through `ultra` is preserved. Verified direct workers treat the runtime as the complete workflow, do not reopen the Skill or duplicate an external verifier, and leave test execution to the configured verifier. Medium uses one combined pre-edit read; high uses at most two in compact workspaces. Medium/high workers prepare the complete edit set and prefer one patch call for two to eight files when supported, without suppressing correction after a failed or incomplete patch; `xhigh`/`ultra` remain unrestricted. Use `--strategy progressive` to pin direct-first escalation, `direct` to forbid escalation, `goal` to plan immediately, `workflow` for one-shot decomposition, or explicit `native` for one bounded read-only parent with native children. Task text cannot select the route or grant capabilities. Without `model-workflow-execute`, a one-shot route stops after full preflight, and direct plan-only spends zero provider calls; adaptive and background routes require that approval. Writes still require `--allow-writes --approve model-workflow-write`. A deterministic command verifier may accept a terminal failed write workspace, but the failed provider run remains visible. Goal evidence records resolved effort, planning mode, and completed planner calls per iteration. The strict routing receipt contains hashes and policy booleans, never task or approval values. Use `list-auto`/`status-auto` or the dashboards to resolve a detached child by its filename/path hash; treat `invalid` or `unresolved` as attention, and use the child run/goal artifact for completion evidence.
- `auto --strategy native` is the explicit Mode 4 fast path. It uses zero planning calls, remains read-only, defaults to one child slot, accepts `--native-agent-threads 1..6`, fixes depth at one, and disables recursive fan-out plus irrelevant plugin/app/browser/MCP/web/hook/goal/memory/dependency surfaces. Current deterministic-v5 product funding is `24,000 + 18,000 * child_slots`, giving the default parent-plus-one-child envelope 42,000 shared weighted tokens. Explicit caller-controlled benchmark caps may go no lower than the legacy `12,000 + 6,000 * child_slots` minimum; deterministic-v4 receipts retain that old formula. `parent-integration-and-synthesis-v4` limits each child to one independent group, requires complementary parent work immediately, stops unfinished child work at half remaining, and starts no-tools synthesis at one-third remaining capped at 6,000 tokens. Synthesis groups cross-method manifestations by invariant/repair site, merges methods or call sites in one component that bypass one required helper, preserves separately repaired components, merges candidates sharing one first faulty line, and retains one missing validation boundary as one finding across clause types before checking the output contract once. It replaces the generic checkpoint clause instead of duplicating it. Repository guidance and the selected model remain intact. Require `--allow-agent --allow-parallel --approve native-agent-delegation`; actual execution also requires `--approve model-workflow-execute`. It rejects writes, adaptive verifiers, maps, and isolated/restricted profiles. Plan-only emits the workflow and v2 routing receipt with zero provider calls. Concurrent provider turns may overshoot before the shared stop is observed; complete aggregate reconciliation fails measured overages closed and retains an answer only as partial failure evidence, while unavailable reconciliation remains advisory.
