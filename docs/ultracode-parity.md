# Ultracode-Style Scope

Codex Conductor targets a bounded subset of orchestration behavior:

- direct model-first execution;
- read-only map, collect, and synthesis;
- explicit permissions and token limits;
- external state and exact-binding resume;
- staged writes with independent verification and explicit apply;
- deterministic receipts and recovery evidence.

Specialist facilities such as team consoles, routines, dashboards, cloud
adapters, and provider comparisons remain optional extras. They are not
imported or initialized by the default runtime.

Capability presence is not quality parity. Cross-provider claims require
matched tasks, models, permissions, budgets, source state, and independent
scoring. No universal superiority claim is supported.
